## 一、核心结论（TL;DR）

**"3倍杠杆 + 三股轮动 + 宽阈值"是经过 1330 组排列组合验证的最优策略。**

> ⚠️ 以下所有「打败B&H」均指**同杠杆 Nx B&H**——公平对比，只比轮动机制是否增值。

| 结论 | 数据支撑 |
|------|----------|
| ❌ 死拿单只股票杠杆 → 不是最优 | 21只股票10年最优杠杆仅 2.0x–3.0x，无人扛住 10x |
| ❌ 两股轮动 → 容错不足 | 卖出资金只能去一个方向，「换错」代价大（BW 78-90%） |
| ❌ 低阈值频繁换仓 → 反亏钱 | 15% 阈值 2025-2026 年换 23 次 → **-47.6%**；50% 仅 2 次 → **+84.5%** |
| ✅ **3x + 三股轮动 + 40-50% 阈值** → 保底最优 | **100%** 年份打败同杠杆最差 B&H（两股最好仅 90%） |
| ⚠️ 轮动核心价值 = 避坑保底 | 三股轮动经常连最好 B&H 也一起打败（70-100% BB） |

## 二、理论基石

### 2.1 波动率衰减公式

杠杆产品的每日重置导致收益偏离简单倍数：

$$\ln\frac{V_T}{V_0} = L \cdot \sum r_i - \frac{L^2}{2} \cdot \sum r_i^2$$

- 第一项是期望收益（线性放大）
- **第二项是衰减项**（随 L² 增长，不可消除）
- 当 L 超过 Kelly 最优值 L* = μ/σ² 时，收益开始下降甚至转负

### 2.2 关键股票 Kelly 最优杠杆

| 股票 | 年化 μ | 年化 σ | Kelly L* | 实际最优 | 10x 结果 |
|------|--------|--------|----------|----------|----------|
| NVDA | 79% | 48% | 3.4x | 2.5x | KO (2022-06) |
| MU | 35% | 52% | 1.3x | 2.0x | KO (2025-04) |
| GOOGL | 28% | 28% | 3.6x | 3.0x | KO (2022-10) |
| AMZN | 31% | 32% | 3.0x | 2.5x | KO (2022-12) |

> **关键发现：理论 Kelly 值比实际最优高出 0.5–2.3x。实操中必须下调杠杆，因为 Kelly 假设连续复利，而真实世界有隔夜跳空、流动性冲击和尾部风险。**

### 2.3 三股 vs 两股轮动的数学优势

```
两股轮动：卖出 A → 全仓买入 B（只有一个方向，可能"换错了"）
三股轮动：卖出 A → 平分给 B 和 C（两个方向，容错率翻倍）

例：A 继续跌、B 涨、C 跌 → 两股换到 B 赚钱，换到 C 亏钱
                        → 三股换到 B+C，至少 B 那半赚钱
```

**核心差异**：三股轮动的「最坏情况」是两股轮动的「平均情况」。

## 三、策略设计

### 3.1 主力策略：3x 三股轮动（★★★★★ 推荐）

```
┌─────────────────────────────────────────────────────────────┐
│  入场：将资金分成 3 等份，每份 $1000                           │
│  标的：MU + ORCL + AMD                                       │
│  杠杆：3x                                                    │
│  规则：每日追踪每个仓位的最高点（peak）                          │
│        当任一仓位从最高点回撤 ≥ 40%                             │
│        → 立即卖出该仓位                                        │
│        → 现金平分给另外两只股票                                 │
│        → 幸存者重置 peak 为当前值                               │
│  退出：不设主动退出，持续轮动                                    │
└─────────────────────────────────────────────────────────────┘
```

**参数选择理由**：
- **MU + ORCL + AMD**：1330 组排列组合 10 年完整数据 **#1**。年均 $33,610（+1020%），10/10 打败最差 BH，7/10 打败最好 BH，**零 KO**
- **3x 杠杆**：收益/风险比最优，10 年零敲出
- **40% 阈值**：在 2025-2026 趋势中不频繁换仓，在 2022 熊市中及时切换
- **仅 ~6 次换仓/年**：低频操作，实操可行

## 五、轮动核心规律

### 5.1 什么样的股票适合轮动？

| 特征 | 适合 | 不适合 | 原因 |
|------|:----:|:------:|------|
| 年化波动率 | > 40% | < 30% | 波动是轮动获利的燃料 |
| 相关性 | 低（< 0.5） | 高（> 0.7） | 同涨同跌 = 轮动失效 |
| 长期趋势 | 任何方向 | 单边碾压 | 太强的股票（NVDA）轮动不如死拿 |
| 行业分散 | 跨行业 | 同行业 | 半导体+软件+电动车 > 纯半导体 |

### 5.2 最佳轮动锚点排名

| 锚点 | 出现次数 (Top 20) | 特点 | 推荐搭档 |
|------|:----------------:|------|----------|
| 🥇 **MU** | 9/20 | 极高波动（52%），适合做锚点 | AMD, ORCL, MRVL, AVGO |
| 🥈 **PLTR** | 14/20 | 数据虽短但效果惊人 | TSLA, MU, ADBE |
| 🥉 **ADBE** | 9/20 | 黑马，SaaS 高波动代表 | MU, PLTR, AMD |
| 4 AMD | 5/20 | 半导体波动担当 | MU, ORCL, MRVL |
| 5 MRVL | 5/20 | 半导体黑马 | MU, AMD, ORCL |

### 5.3 哪些股票不该放进轮动？

| 股票 | 问题 | 在 Top 20 出现 |
|------|------|:-------------:|
| **MSFT** | 波动太低（~28%），没有切换机会 | 0 次 |
| **GOOGL** | 波动太低（~28%），跟谁都同涨同跌 | 1 次 |
| **NVDA** | 太强了，死拿比轮动更赚钱 | 1 次 |
| **QCOM** | 波动一般，没有突出表现 | 0 次 |

### 5.4 阈值选择铁律

| 市场状态 | 推荐阈值 | 原因 |
|----------|:--------:|------|
| 强趋势牛市 | **40-50%** | 宽阈值避免被小回调震出去 |
| 震荡分化 | 30-40% | 给轮动留空间但不频繁换仓 |
| 熊市普跌 | 40-50% | 熊市中频繁换仓 = 反复被收割 |
| 极端波动 | 30-40% | 稍微灵敏一点捕捉切换时机 |

> ⚠️ **核心原则**：阈值宁可宽不可窄。15-30% 阈值在 2025-2026 年反亏钱（换仓 23 次 → -47.6%），而 50% 仅换 2 次 → +84.5%。

## 六、风险管理体系

### 6.1 绝对不能做的事

| 禁止事项 | 原因 |
|----------|------|
| 🚫 使用 5x 以上杠杆 | 10 年间每只股票都有敲出记录 |
| 🚫 单吊一只股票加杠杆 | 2022 NVDA -50%，单吊 3x 直接 -90% |
| 🚫 用两股轮动代替三股 | 两股 BW 最多 90%，三股 100%；两股有「换错」风险 |
| 🚫 用低阈值（≤ 20%）在趋势市中 | 频繁换仓 = 反复被小回调收割 |
| 🚫 纯同行业组合 | 半导体+半导体+半导体，一荣俱荣一损俱损 |
| 🚫 把 MSFT/GOOGL 放进轮动 | 波动太低，轮动退化为持仓不动 |

### 6.2 风控规则

```
┌──────────────────────────────────────────────────────────────┐
│  1. 仓位上限：单组合不超过总资金 50%                            │
│  2. 组合止损：如果组合整体亏损超 30%（从最高点），暂停交易 1 个月  │
│  3. 年度审视：每年 6 月回测一次，调整阈值（±10%）                │
│  4. 现金储备：始终保持 ≥ 20% 现金，不在极端行情满仓               │
│  5. 杠杆上限：任何情况下单标的杠杆 ≤ 3x                         │
│  6. 标的审视：如果某只股票连续 2 年波动率降至 < 30%，替换它       │
│  7. 组合分散：至少一组配置含不同行业的股票                        │
└──────────────────────────────────────────────────────────────┘
```

### 6.3 极端情景压力测试

| 情景 | 历史参照 | 预期影响 | 应对 |
|------|----------|----------|------|
| MU 暴跌 44% | 2018-2019 已发生 | 3x MU 跌 ~90%，触发阈值 → 平分到 ORCL+AMD | 40% 阈值及时切换 |
| 半导体集体暴跌 | 2022 科技股普跌 | 全半导体组合可能全部触发阈值 | 跨行业组合更安全 |
| 三股同时触发阈值 | 极端情况（罕见） | 全部卖出 → 平分 → 重新开始追踪 | 相当于「硬重置」 |
| ORCL+MSFT+AMZN 平缓下跌 | 2018-2019 | 阈值宽（50%）可能不触发，跟随下跌 | 这是三股唯一的软肋——阈值太宽 = 不止损 |
| 连续敲出 | 5x 杠杆才出现 | 3x 在 MU+ORCL+AMD 上零 KO | **3x 是唯一的保护** |

## 附录 B：Score 计算公式

```
Score = (Rotation_Avg - Worse_BH_Avg) / Worse_BH_Avg × 100

其中：
  Rotation_Avg = 轮动在最佳阈值下的幸存年份平均终值
  Worse_BH_Avg = 三只股票中同杠杆 B&H 年均终值最低的那只

解释：Score 衡量的是轮动相比「踩到最差 B&H 雷」多赚了多少。
      Score 越高 = 轮动避坑效果越显著。
```

## 附录 C：术语表

| 术语 | 定义 |
|------|------|
| **BW** (Beat Worse) | 轮动收益 > 同杠杆下三只 B&H 中最差的那只 |
| **BB** (Beat Better) | 轮动收益 > 同杠杆下三只 B&H 中最好的那只 |
| **KO** (Knock Out) | 总资产 ≤ 初始的 5%（$150），视为爆仓 |
| **Score** | 轮动超额收益 vs 最差 B&H 的百分比 |
| **10Y clean** | 10 年完整数据，不含 PLTR（仅 6 年数据） |
| **Peak** | 每个仓位的日内最高点，每日滚动更新 |
| **阈值** | 触发轮动的回撤幅度（从 peak 算起） |
